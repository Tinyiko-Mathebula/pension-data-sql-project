Project Title
Pension Data Management and Contribution Analysis

Project Overview
This project simulates a pension administration system used to manage member data, pension schemes, and contribution records. The goal of the project is to demonstrate data management, data quality auditing, and financial contribution analysis using SQL.

Objectives
• Design a relational database for pension administration
• Store member, employer, scheme, and contribution data
• Perform data quality checks to identify missing or incomplete records
• Analyse pension contributions and scheme performance
• Prepare datasets suitable for management reporting and dashboards

Tools Used
MySQL Workbench – Database design and SQL analysis
SQL – Data querying and reporting
CSV – Data export for dashboard analysis

Database Structure
The database includes the following tables:
• schemes – stores pension scheme information
• employers – stores employer information
• members – stores pension member biodata
• contributions – stores member contribution transactions
• transfers – tracks scheme transfers

Key Analysis Performed
• Identification of members with missing contact information
• Detection of members with no contributions
• Calculation of total contributions per member
• Calculation of total contributions per pension scheme
• Creation of a dashboard dataset for reporting and visualization

Key Insights
The analysis identified members with incomplete records and members who had not made pension contributions. This information can help pension administrators follow up with employers and improve data quality and compliance.

Conclusion
This project demonstrates practical SQL skills in data management, reporting, and financial data analysis within a pension administration context.
